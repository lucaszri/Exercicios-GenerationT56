1. O que são métodos HTTP?
R: É um protocolo de requisição e resposta.

2. Quais são os métodos HTTP mais utilizados atualmente?
R: GET, POST, PUT e DELETE

3. Quais são os 2 principais métodos do HTTP? Qual é a diferença entre eles? 
R: GET e POST. GET manda as informações para servidor 

4. O que são requisitações HTTP? Quais os tipos? 
R: Requisições HTTP são mensagens enviadas pelo cliente para iniciar uma ação no servidor. São do tipo GET, POST, HEAD, OPTIONS, PUT, DELETE e TRACE.

5. Como funciona uma requisição?
R: Cliente vai utilizar algum dos métodos HTTP para fazer uma requisição e o servidor da uma resposta, protocolo Request-Response. 

6. O que é um método GET?
R: O método GET solicita a representação de um recurso específico. Requisições utilizando o método GET devem retornar apenas dados.

7. Qual é o protocolo de internet para transferência segura, com uso de certificado digital, utilizado em sites de compras eletrônicas?
R: HTTPS significa HyperText Transfer Protocol Secure (Protocolo de Transferência de Hipertexto Seguro) e é usado para criptografia e autenticação de comunicação entre o servidor web e o navegador.

8. Quais os principais métodos de uma requisição REST?
R: GET, POST, PUT e DELETE só que podem ser requisitadas de forma diferente. 

9. Qual a diferença entre IPv4 e IPv6?
R: O Protocolo de Internet versão 4 possui endereços no padrão 32 bits e sustenta cerca de 4 bilhões de combinações de endereços IP em todo o mundo. Já o IPv6 é a versão 6 do Protocolo de Internet, com endereços no padrão 128 bits, permitindo, portanto, cerca de 340 undecilhões de endereços

10. Faça um resumo sobre os seguintes protocolos: http, tcp, ip, udp, smtp e ftp.

HTTP: Um protocolo de transferência que possibilita que as pessoas que inserem a URL do seu site na Web possam ver os conteúdos e dados que nele existem.

TCP: É responsável pela entrega dos dados assim que o endereço IP for encontrado.

IP: É a parte que obtém o endereço para o qual os dados são enviados.

UDP: É um protocolo de Internet mais simples e sem conexão, no qual os serviços de verificação e recuperação de erros não são necessários.

SMTP: Protocolo de Transferência de Correio Simples é a tecnologia que permite que e-mails sejam enviados de um servidor para outro até serem entregues na caixa de correio.

FTP:  Protocolo de Transferência de Arquivos.
